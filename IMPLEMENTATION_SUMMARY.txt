================================================================================
KEY FEATURES
================================================================================

UNIFIED ORDER FORMAT
{
    "source": "yampi" | "appmax",
    "order_number": "string",
    "status": "pago|nao_pago|pendente|cancelado|recusado|abandonado|reembolsado|enviado|entregue",
    "status_label": "Portuguese label",
    "total": float,
    "payment_method": "string or null",
    "created_at": "ISO timestamp",
    "tracking_codes": [
        {"code": "ABC123", "status": "enviado", "url": "..."}
    ],
    "items": [
        {"name": "Product", "quantity": 1, "price": 99.90}
    ],
    "raw": {} // Original API response
}

STATUS NORMALIZATION (Both Platforms)
- pago → Paid
- nao_pago → Not Paid
- pendente → Pending
- cancelado → Cancelled
- recusado → Declined
- abandonado → Abandoned
- reembolsado → Refunded
- enviado → Shipped
- entregue → Delivered

CONFIGURATION PRIORITY
1. Environment variables (highest priority)
2. JSON file (backend/app/config/ecommerce.json)
3. Database (optional, via migration)

SECURITY FEATURES
- Credentials never logged
- API keys masked in responses (****...****)
- All external calls via HTTPS
- JWT authentication required
- No sensitive data in logs
- 15-second timeout per request

SETUP INSTRUCTIONS
================================================================================

1. ADD ENVIRONMENT VARIABLES
   export YAMPI_TOKEN=your_token_here
   export YAMPI_ALIAS=your_store_alias
   export APPMAX_API_KEY=your_api_key_here

2. (OPTIONAL) RUN DATABASE MIGRATION
   psql -U carbon -d carbon_helpdesk < backend/migrations/001_ecommerce_settings.sql

3. RESTART APPLICATION
   docker-compose restart backend
   # or
   python -m uvicorn app.main:app --reload

4. VERIFY INSTALLATION
   curl http://localhost:8000/api/health
   # Should return {"status": "ok", "service": "Carbon Helpdesk"}
